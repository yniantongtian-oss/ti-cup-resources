# RS485 / Modbus RTU 联调指南

适用场景：STM32F103C8T6 + MAX485 下位机，通过 USB 转 RS485 与电脑上位机通信。

## 推荐起始配置

- 物理层：两线半双工 RS485
- 波特率：9600 bps
- 数据格式：8N1
- 从站地址：1
- 终端电阻：仅在线缆两端各 120 Ω
- 偏置电阻：总线上只保留一组，避免多组偏置互相拉扯
- 地线：短距离低压联调时连接参考地；长距离或高共模环境应使用隔离收发器

## 接线检查

1. A 对 A、B 对 B；不同厂商对 A/B 命名可能相反，通信失败时先查芯片数据手册。
2. MAX485 的 DE 与 `/RE` 可并联，由一个 GPIO 控制。
3. 发送前拉高 DE，等待 UART TC 置位后再拉低，不能只等 TXE。
4. USB 转 RS485 的自动收发模块通常无需控制 DE。
5. 总线空闲电压应稳定，不能悬空或持续振荡。

## 最小联调顺序

### 1. 回环验证

先不接功率级，仅让 STM32 周期性返回固定寄存器：

- 固件版本：`0x0100`
- 状态：IDLE
- 母线电压：固定测试值

### 2. CRC 验证

仓库工具可以生成标准帧：

```bash
python tools/modbus_rtu.py read 1 0 10
python tools/modbus_rtu.py write 1 1 1000
```

已知读 10 个保持寄存器请求：

```text
01 03 00 00 00 0A C5 CD
```

### 3. 示波器/逻辑分析仪检查

- A/B 差分波形是否存在
- 发送期间 DE 是否保持有效
- 最后一个停止位结束后 DE 是否及时释放
- 响应间隔是否符合 Modbus RTU 至少 3.5 个字符时间的帧间隔要求

### 4. 上位机检查

记录以下参数，避免“软件没问题但配置不同”：

- COM 口
- 波特率
- 校验位
- 停止位
- 从站地址
- 功能码
- 寄存器地址是 0 基还是 1 基
- 数值缩放比例和有符号类型

## 常见故障

| 现象 | 优先检查 |
|---|---|
| 完全无响应 | A/B 接反、DE 未拉高、从站地址错误、UART 引脚错误 |
| 偶发 CRC 错误 | DE 释放过早、波特率误差、线缆干扰、终端/偏置错误 |
| 值差 10/100/1000 倍 | 寄存器缩放比例不一致 |
| 负数变成大正数 | 上位机按 uint16 解析了 int16 |
| 第一帧正常后无响应 | `/RE` 未恢复接收、发送完成判断错误 |
| 多节点冲突 | 地址重复或有多个主站同时发送 |

## 安全要求

通信联调必须与功率输出解耦。建议上位机建立以下规则：

- 连接后默认只读，不自动 Arm
- 写入参考值不等于启动
- 通信超时后下位机自行 Disarm
- 急停不能只依赖 Modbus 命令
- 故障清除前必须先确认测量值恢复正常
