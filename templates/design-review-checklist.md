# 竞赛项目设计评审清单

项目：__________　版本：__________　日期：__________　评审人：__________

## 一、需求与评分点

- [ ] 已逐条拆解题目要求、基础指标、发挥指标和扣分项
- [ ] 每个指标都有对应测量方法、仪器、工况和通过标准
- [ ] 明确不能使用的器件、供电、尺寸、成本或通信方式
- [ ] 评分展示顺序已经设计，现场可以在规定时间内完成
- [ ] 关键参数单位、有效值/峰值、稳态/瞬态定义无歧义

## 二、系统架构

- [ ] 功率路径、控制路径、采样路径和通信路径分离清楚
- [ ] 每个模块都有输入、输出、量程、带宽和故障行为
- [ ] MCU 算力、Flash、RAM、ADC、定时器和串口资源足够
- [ ] 关键模块存在降级方案或可快速替换方案
- [ ] 上电、复位、失联和故障后的默认状态安全

## 三、硬件

- [ ] 原理图完成 ERC/人工复核，网络名和极性一致
- [ ] 电源树标出最大电流、纹波、启动顺序和散热
- [ ] MOSFET/二极管/电感/电容额定值留有合理裕量
- [ ] 模拟地、功率地、数字地和采样回流路径清晰
- [ ] 栅极回路、开关节点和高 di/dt 环路面积最小化
- [ ] ADC 输入具有限流、滤波和过压保护
- [ ] 过流、过压、过温、短路和反接保护均有设计
- [ ] 急停、保险丝和硬件关断不依赖软件
- [ ] 调试点、测试夹位置和丝印方向方便现场测量
- [ ] BOM 中无停产、难采购或交期不可控的关键器件

## 四、固件

- [ ] 状态机明确：初始化、待机、运行、故障、升级
- [ ] PWM 上电为 0，解锁条件完整，正反向不会同时导通
- [ ] ADC/DMA 数据具有标定、滤波、限幅和有效性检查
- [ ] 控制周期由硬件定时，最坏执行时间低于周期预算
- [ ] PI/PID 参数有来源、调参记录和饱和处理
- [ ] 通信超时自动停机，错误帧不会改变控制状态
- [ ] 故障锁存、故障码、清除条件和日志完整
- [ ] Flash 参数有版本、范围检查、CRC 和默认回退
- [ ] 编译警告视为错误，至少有核心逻辑单元测试

## 五、通信与上位机

- [ ] 协议版本、字节序、有符号数、缩放和地址已冻结
- [ ] RS485 方向切换等待 UART TC，不是仅等待 TXE
- [ ] 上位机连接后默认只读，不自动启动设备
- [ ] 实时曲线、状态、故障、原始值和单位显示一致
- [ ] 日志包含时间戳、设备地址、命令、响应和异常
- [ ] 断线重连、串口占用、CRC 错误和设备重启已测试

## 六、测试与证据

- [ ] 已建立测试用例编号和需求追踪关系
- [ ] 每项结果记录环境、仪器、固件版本和原始数据
- [ ] 先完成无功率、低压限流，再逐级提升工况
- [ ] 极限值附近完成重复性、温升和长时间稳定性测试
- [ ] 失败测试保留记录并形成原因、修复和回归结果
- [ ] 图表可以从原始 CSV/日志重新生成

## 七、答辩与现场

- [ ] 备机、备板、保险丝、线材、转接头和下载器齐全
- [ ] 演示脚本包含正常流程、发挥指标和故障保护展示
- [ ] 关键结果可在 30 秒内向评委复现
- [ ] PPT 参数与实物/报告/代码一致
- [ ] 现场无网络时仍可编译、烧录、查看文档和演示
